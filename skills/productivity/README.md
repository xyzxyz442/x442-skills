# productivity

Skills for daily non-code workflow: writing, communicating, and moving work along. Use these
when the task is about _producing or shaping output for people_ — an announcement, a summary,
a document — rather than about building or reasoning about code itself. For the latter, see
[engineering](../engineering/README.md).

## Skills in this category

| Skill                  | Status         | Purpose                                                                                                                                                                                                    |
| ---------------------- | -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `release-announcement` | `experimental` | Turn a tagged version and its changelog into a user-facing announcement shaped for its channel (GitHub release, Slack, email), leading with user impact rather than the file diff. Multi-language capable. |

Full per-skill detail (prerequisites, verification harness, status meanings) lives in the
[skills catalog](../README.md).
